# Skill: AI-Driven Dev Architect (SOP v2.0)

# 第一阶段：需求对齐与蓝图 (Alignment & Blueprint)
目标：防止 AI 瞎写，确保它完全理解业务逻辑。
投喂背景：将需求文档、Jira 票据或口头需求发给 AI。
逻辑核对 (Prompt)：
"你看对于这个功能，请你还有什么不明白的或者有什么问题，提出来。
为了确保你完全理解，请做两件事：
列一条具体的场景示例（Input -> Processing -> Output），供我核对你的逻辑是否无误。
输出一段伪代码 (Pseudo-code) 或 Mermaid 时序图，展示核心业务流。
确认清晰且没问题之后，请列一个详细的开发任务列表 (Task List)。"
人工确认： 检查场景和伪代码是否符合预期。
注：这里最好让ai 生成文档，往后所有的对话都引用这个文档

# 第二阶段：开发与测试固化 (Coding & Locking)
目标：先跑通，并用单测锁死逻辑，为重构做准备。
代码生成：让 AI 执行开发任务。
人工验证： 手动简单测试，确保主流程能跑通。
生成防守性测试 (Prompt)：
"我需要你切换视角，用 git diff 以代码逻辑覆盖的角度来看这个改动。
请为这段代码编写单元测试 (JUnit/TestNG)：
覆盖率：必须包含正常路径 (Happy Path) 和异常路径 (Sad Path)。
边界测试：重点测试边界值（如空列表、0、负数、超长字符串、最大数值）。
断言：请确保 assert 校验了核心业务结果。"
运行测试： 运行测试用例。必须全绿 (Pass) 才能进入下一阶段。

# 第三阶段：安全重构 (Safe Refactoring)
目标：在有测试保护的情况下优化代码结构。
执行重构 (Prompt)：
"现在测试已经通过。请你重构刚才的代码，但必须保证原逻辑不变（依托刚才的单测保证）。
重构要求：
结构：避免深度嵌套的 if-else，优先使用卫语句（Return Early）。
复用：提取重复逻辑为独立函数 (Private Methods)。
命名：变量名使用有业务含义的名称，严禁使用 A, B, Temp 等。
Lombok：合理使用 @Builder 或 @Data 简化样板代码。
重构后，请简述你优化了哪些点。"
回归测试： 再次运行第二阶段的测试。如果报错，立即回滚或修复。

# 第四阶段：深度代码审查 (Deep Code Review)
目标：模拟 Tech Lead 视角，查缺补漏。
准备上下文：
如果用 Cursor/Copilot：打开相关联的 DTO, Service, Entity 文件。
如果用网页版：复制 git diff 加上核心调用函数的代码。
执行 CR (Prompt)：
"你可以通过 git diff (以及我提供的上下文) 来访问我的修改吗？
角色：请切换身份，变成一名以“毒舌”和“严谨”著称的 Senior Tech Lead。 任务：对上述代码进行 Code Review，请忽略缩进、换行等主观审美问题，重点审查以下致命项：
业务逻辑漏洞：我的使用场景是 [cpa会用这个页面去州政府填报税务数据]，是否存在逻辑死角、数据精度丢失或计算错误？
规范性：是否完全遵守了 阿里巴巴 Java 开发手册 和 i18n 要求？
安全性：是否存在 SQL 注入、XSS、越权风险 (IDOR)？
性能：是否存在循环查库 (N+1)、内存泄漏、大对象未分页风险？
输出格式： 如果代码完美，请仅回复“LGTM (Looks Good To Me)”。 否则，请依据危险程度由高到低列出修改清单。"
人工修正： 针对 AI 提出的 Critical/Major 问题进行修改（忽略它的过度吹毛求疵）。

# 第五阶段：提交与发布 (Commit & PR)
目标：自动化生成文档，优雅收尾。
生成 PR 信息 (Prompt)：
"我已经修正完上述 bug 并通过测试。现在请切换身份，假如你是一名规范的程序员：
分支命名：你会如何新建分支？(参考格式：xjl/feat-orderReport-levelTaxRate)
Commit Message：请遵循 Conventional Commits 规范 (feat: xxx / fix: xxx)。
PR 描述：请写一份清晰的 PR 描述，包含变动摘要、测试截图占位符、以及特别注意事项。"

# 紧急情况处理 (Troubleshooting)
死循环法则：当 AI 在某个逻辑点卡住、开始车轱辘话重复，或者连续两次修改都无法通过测试时：
立刻停止对话。
新建对话窗口 (New Chat)。
或者切换模型 (如从 Claude 4.5 Sonnet 切到 gemini，或反之)。
重新把当前代码和报错信息丢进去，不要纠结于之前的上下文。
